## `StandardMode.vue` - 标准模式页面

### 📋 基本信息

**路径**: `src/views/StandardMode.vue`  
**路由**: `/`  
**作用**: 应用的主页面，显示和控制所有设备

### 🎯 主要功能

#### 1. 显示设备卡片
- 显示当前房间的所有设备
- 每个设备显示：名称、类型、状态（在线/离线）
- 如果有定时器，显示倒计时

#### 2. 设备交互
- **单击设备卡片**: 显示底部控制栏
  - 灯光设备 → 显示亮度控制（在 App.vue 中）
  - 开关/路由器 → 显示电源按钮
  - 洗衣机/烘干机 → 显示洗衣控制栏
  - 扫地机器人 → 显示机器人控制栏
  - 晾衣架 → 显示晾衣架控制栏

- **长按设备卡片**: 打开设备详情对话框
  - 插座设备 → 打开插座对话框
  - 其他设备 → 打开通用设备对话框

- **拖拽滚动**: 左右拖动查看更多设备

#### 3. 房间切换
- 顶部导航栏显示所有房间
- 点击房间标签切换显示的设备

#### 4. 定时器管理
- 实时更新定时器倒计时
- 显示最近的定时器（定时开启/关闭/倒计时）
- 定时器触发时显示通知

### 🔧 核心逻辑

#### 设备过滤
```typescript
const filteredDevices = computed(() => {
  const currentTab = tabsStore.tabs[tabsStore.activeIndex]
  return devicesStore.devices.filter(d => d.location === currentTab)
})
```
- 根据当前房间过滤设备
- 只显示当前房间的设备

#### 定时器更新
```typescript
const updateTimerRemaining = () => {
  devicesStore.devices.forEach(device => {
    if (device.timerOffEnabled) {
      timerOffRemainingMap.value[device.id] = 
        devicesStore.getTimerRemaining(device.id, 'off')
    }
    // ... 更新其他定时器
  })
}

// 每秒更新一次
setInterval(updateTimerRemaining, 1000)
```

#### 长按检测
```typescript
const LONG_PRESS_DURATION = 500  // 500毫秒

const onCardPressStart = (device: Device) => {
  longPressTimer.value = setTimeout(() => {
    // 打开对话框
    selectedDevice.value = device
    showDeviceDialog.value = true
  }, LONG_PRESS_DURATION)
}

const onCardPressEnd = () => {
  if (longPressTimer.value) {
    clearTimeout(longPressTimer.value)
  }
}
```

#### 拖拽滚动
```typescript
const onMouseDown = (e: MouseEvent) => {
  isDragging.value = true
  startX.value = e.pageX
  scrollLeft.value = cardsContainer.value.scrollLeft
}

const onMouseMove = (e: MouseEvent) => {
  if (!isDragging.value) return
  const x = e.pageX
  cardsContainer.value.scrollLeft = 
    scrollLeft.value - (x - startX.value) * 1.5
}
```

### 🎨 样式特点

### 🔗 事件流

## `AddDevice.vue` - 添加设备页面

### 📋 基本信息

**路径**: `src/views/AddDevice.vue`  
**路由**: `/equipment`  
**作用**: 管理设备，添加新设备，查看所有设备

### 🎯 主要功能

#### 1. 添加设备表单
- 输入设备名称
- 选择设备类型（照明、插座开关、清洁电器等）
- 选择子类型（根据主类型动态显示）
- 选择设备位置（房间）
- 点击"确认添加"按钮添加设备

#### 2. 设备列表
- 显示所有已添加的设备
- 每个设备显示：名称、类型、位置、状态
- 提供开关按钮
- 提供删除按钮

#### 3. 表单验证
- 检查设备名称是否为空
- 检查是否选择了位置
- 检查是否重复添加（相同名称、类型、子类型）

### 📦 组件结构


### 🔧 核心逻辑

#### 添加设备
```typescript
const addDevice = () => {
  // 1. 验证表单
  if (!formData.value.name || !formData.value.location) {
    return
  }
  
  // 2. 检查重复
  const isDuplicate = devicesStore.devices.some(
    d => d.name === formData.value.name && 
         d.type === formData.value.type &&
         getDeviceSubType(d) === getCurrentSubType()
  )
  
  if (isDuplicate) {
    ElMessage.warning('该设备已存在，请勿重复添加')
    return
  }
  
  // 3. 添加设备
  devicesStore.addDevice({
    id: Date.now().toString(),
    ...formData.value,
    status: 'online'
  })
  
  // 4. 重置表单
  formData.value = { ... }
  showForm.value = false
  
  // 5. 跳转到主页面
  router.push('/')
}
```

#### 获取子类型
```typescript
const getCurrentSubType = () => {
  switch (formData.value.type) {
    case 'light': return formData.value.lightType
    case 'switch': return formData.value.switchType
    case 'cleaner': return formData.value.cleanerType
    // ... 其他类型
    default: return ''
  }
}
```

#### 获取类型标签
```typescript
const getTypeLabel = (type: string) => 
  getLabelByValue(DEVICE_TYPES, type)

const getLightTypeLabel = (lightType: string) => 
  getLabelByValue(LIGHT_TYPES, lightType)

// ... 其他类型标签获取函数
```

## 💡 使用建议

### 修改设备卡片样式
1. 打开 `StandardMode.vue`
2. 找到 `.device-content` 样式
3. 修改 CSS

### 添加新的设备类型
1. 在 `core/constants/deviceTypes.ts` 添加类型标签
2. 在 `core/constants/deviceOptions.ts` 添加选项
3. 在 `AddDevice.vue` 添加子类型选择表单项

### 修改设备卡片交互
1. 打开 `StandardMode.vue`
2. 找到 `onCardClick` 函数
3. 添加新的设备类型判断和处理逻辑

### 自定义底部控制栏
1. 在 `features/device/components/` 创建新组件
2. 在 `StandardMode.vue` 中导入
3. 在 `onCardClick` 中添加显示逻辑
