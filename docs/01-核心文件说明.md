# 核心文件详细说明

## 📄 根目录文件

### `src/App.vue` - 应用根组件

**作用**: 整个应用的最外层容器，所有页面的父组件

**主要功能**:
1. **顶部信息栏**
   - 显示时钟（日期和时间）
   - 显示天气信息
   
2. **模式切换按钮**
   - 在标准模式和添加设备模式之间切换
   
3. **路由视图**
   - 显示当前路由对应的页面
   - 标准模式页面或添加设备页面
   
4. **底部亮度控制**
   - 选中灯光设备后显示
   - 控制灯光亮度
   - 切换灯光开关

**关键代码结构**:
```vue
<template>
  <div class="app-container">
    <!-- 顶部信息 -->
    <div class="top-section">
      <Clock />
      <Weather />
    </div>
    
    <!-- 模式切换 -->
    <ModeBadge />
    
    <!-- 页面内容 -->
    <div class="middle-section">
      <router-view @select-light="onSelectLight" />
    </div>
    
    <!-- 底部亮度控制 -->
    <div class="bottom-section">
      <BrightnessControl 
        :device="selectedLightDevice"
        @update:brightness="onUpdateBrightness"
        @toggle-power="onTogglePower"
      />
    </div>
  </div>
</template>
```

**状态管理**:
- `selectedLightDevice`: 当前选中的灯光设备
- 监听路由变化，自动关闭亮度控制
- 监听点击外部，关闭亮度控制

**事件处理**:
- `onSelectLight`: 选择灯光设备
- `onUpdateBrightness`: 更新亮度
- `onTogglePower`: 切换电源
- `handleClickOutside`: 点击外部关闭控制

### `src/main.ts` - 应用入口文件

**作用**: 应用的启动入口，初始化所有配置

**主要功能**:
1. **创建 Vue 应用实例**
   ```typescript
   const app = createApp(App)
   ```

2. **安装 Pinia（状态管理）**
   ```typescript
   app.use(createPinia())
   ```
   - 用于管理设备数据、房间标签等全局状态

3. **安装 Vue Router（路由）**
   ```typescript
   app.use(router)
   ```
   - 用于页面切换

4. **安装 Element Plus（UI 组件库）**
   ```typescript
   app.use(ElementPlus)
   ```
   - 提供表单、对话框等 UI 组件

5. **导入全局样式**
   ```typescript
   import '@/styles/style.css'
   ```

6. **挂载应用到 DOM**
   ```typescript
   app.mount('#app')
   ```

**完整代码**:
```typescript
import { createApp } from 'vue'
import { createPinia } from 'pinia'
import ElementPlus from 'element-plus'
import 'element-plus/dist/index.css'

import App from './App.vue'
import router from './router'
import '@/styles/style.css'

const app = createApp(App)

app.use(createPinia())
app.use(router)
app.use(ElementPlus)

app.mount('#app')
```

## 🛣️ 路由配置

### `src/router/index.ts` - 路由定义

**作用**: 定义应用的所有路由规则

**路由列表**:

| 路径 | 名称 | 组件 | 说明 |
|------|------|------|------|
| `/` | standard | StandardMode.vue | 标准模式（主页面） |
| `/equipment` | equipment | AddDevice.vue | 添加设备页面 |

**代码结构**:
```typescript
import { createRouter, createWebHistory } from 'vue-router'
import StandardMode from '@/views/StandardMode.vue'
import AddDevice from '@/views/AddDevice.vue'

const router = createRouter({
  history: createWebHistory(import.meta.env.BASE_URL),
  routes: [
    {
      path: '/',
      name: 'standard',
      component: StandardMode
    },
    {
      path: '/equipment',
      name: 'equipment',
      component: AddDevice
    }
  ]
})

export default router
```

**路由模式**:
- 使用 HTML5 History 模式
- URL 不带 `#` 号
- 需要服务器配置支持

**使用方式**:
```typescript
// 在组件中使用
import { useRouter } from 'vue-router'

const router = useRouter()
router.push('/equipment')  // 跳转到添加设备页面
router.push('/')           // 跳转到主页面
```

## 📝 配置文件

### `vite.config.ts` - Vite 构建配置

**作用**: 配置 Vite 构建工具

**主要配置**:

1. **路径别名**
   ```typescript
   resolve: {
     alias: {
       '@': fileURLToPath(new URL('./src', import.meta.url))
     }
   }
   ```
   - 可以使用 `@/` 代替 `src/`
   - 例如: `import { xxx } from '@/core/constants'`

2. **开发服务器配置**
   ```typescript
   server: {
     allowedHosts: [
       '.trycloudflare.com',
       '.ngrok-free.app',
       '.cpolar.top'
     ]
   }
   ```
   - 允许特定域名访问
   - 用于内网穿透测试

3. **插件配置**
   ```typescript
   plugins: [
     vue()  // Vue 3 支持
   ]
   ```

### `tsconfig.json` - TypeScript 配置

**作用**: 配置 TypeScript 编译选项

**主要配置**:
- 启用严格模式
- 配置路径别名
- 配置模块解析

### `package.json` - 项目配置

**作用**: 定义项目依赖和脚本

**主要脚本**:
```json
{
  "scripts": {
    "dev": "vite",              // 启动开发服务器
    "build": "vite build",      // 构建生产版本
    "preview": "vite preview"   // 预览生产版本
  }
}
```

**主要依赖**:
- `vue`: Vue 3 框架
- `vue-router`: 路由管理
- `pinia`: 状态管理
- `element-plus`: UI 组件库
- `typescript`: TypeScript 支持
- `vite`: 构建工具

## 💡 使用建议

### 修改全局样式
1. 打开 `src/styles/style.css`
2. 修改 CSS 变量或添加新样式
3. 所有页面自动应用

### 添加新路由
1. 打开 `src/router/index.ts`
2. 在 `routes` 数组中添加新路由
3. 创建对应的页面组件

### 修改路径别名
1. 打开 `vite.config.ts`
2. 在 `resolve.alias` 中添加新别名
3. 在 `tsconfig.json` 中同步配置

### 添加新的全局插件
1. 打开 `src/main.ts`
2. 导入插件
3. 使用 `app.use()` 安装
